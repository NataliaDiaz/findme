Overview

The WLLocationManager class is your way of getting access to individual location data from the Walkbase Analytics service.

The WLLocationManager provides support for the following features:

  • Tracking any changes in the user’s position within the confines of a Walkbase Analytics deployment.
  • Fetching the last known location of the user within the confines of a Walkbase Analytics deployment.

Warning: Note: In addition to the other requirements, the user must have WiFI enabled on his/her device or Walkbase Location services will not be available.

Initiating location updates

  • – startUpdatingLocation

    Start updating the user’s location. This will generate callbacks to the locationManager:didUpdateToLocation:fromLocation: delegate method. If anything fails, the locationManager:didFailWithError: method will be called.
  • – stopUpdatingLocation

    Stop updating location, closing the connection to the Walkbase API.

Getting last known location

pausesLocationUpdatesAutomatically

A Boolean value indicating whether the location manager object may pause location updates.

@property (nonatomic) BOOL pausesLocationUpdatesAutomatically

Discussion

Allowing the location manager to pause updates can improve battery life on the target device without sacrificing location data. When this property is set to YES, the location manager pauses updates (and powers down the appropriate hardware) at times when the location data is unlikely to change. Defaults to YES.
