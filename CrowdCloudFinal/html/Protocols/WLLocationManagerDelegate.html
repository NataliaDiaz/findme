WLLocationManagerDelegate Protocol Reference

Declared in WLLocationManager.h

Overview

The WLLocationManagerDelegate protocol defines the methods used to receive location updates from a WLLocationManager object.

Upon receiving a successful location or heading update, you can use the result to update your user interface or perform other actions. If the location could not be determined, you might want to stop updates for a short period of time and try again later. You can use the stopUpdatingLocation method of WLLocationManager to stop location updates.

locationManager:didFailToFetchLastKnownUserLocationWithError:

Tells the delegate that the location manager was unable to retrieve a location value.

- (void)locationManager:(WLLocationManager *)manager didFailToFetchLastKnownUserLocationWithError:(NSError *)error

Parameters

manager

The location manager object that was unable to retrieve the location.

error

The error object containing the reason the location could not be retrieved.

Discussion

Implementation of this method is optional. You should however implement this method. There are a few possible causes for this error to occur, including:

  • The device has no available internet connection.
  • The library can not reach the Walkbase Analytics servers.
  • The library can not read the MAC address of the phone.
  • You are using an invalid API key.
  • The user has never shown up at any Walkbase Analytics site which your API key is allowed to access.

locationManager:didFetchLastKnownUserLocation:

Tells the delegate that the location manager fetched the last known (to us) user location

- (void)locationManager:(WLLocationManager *)manager didFetchLastKnownUserLocation:(WLLocation *)location

Parameters

manager

The location manager object that fetched the location.

location

The location that was retrieved.

Discussion

Implementation of this method is optional. You should however implement this method. This method can be used to determine when the user has been spotted last at a Walkbase Analytics location. Calling this method will fetch the last known location for the calling iPhone/iPad from all the locations available to the calling user’s API key.

locationManager:didUpdateToLocation:fromLocation:

Tells the delegate new location data is available.

- (void)locationManager:(WLLocationManager *)manager didUpdateToLocation:(WLLocation *)newLocation fromLocation:(WLLocation *)oldLocation

Parameters

manager

The WLLocationManager that fired the event.

newLocation

The newest location available via the Walkbase Analytics service.

oldLocation

The previous location, useful for extrapolating which way the user is heading or the user’s speed.

Discussion

Implementation of this method is optional. You should however implement this method. The method delivers location updates in real-time as soon as they are calculated on the Walkbase Analytics back-end. You can use the [WLLocation timestamp] property to determine how “fresh” the location is.
